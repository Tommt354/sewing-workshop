# Швейний цех — облік крою, швей та виплат

Веб-платформа для керування швейним цехом: облік крою по моделях і розмірах, кабінет швеї для взяття/здачі робіт, виплати по тижнях, аналітика собівартості.

## Стек

- **Next.js 14** (App Router) + TypeScript
- **PostgreSQL** + **Prisma ORM**
- **Tailwind CSS** для стилів
- **JWT** в HTTP-only cookies для авторизації
- **bcrypt** для хешування PIN-кодів

## Локальний запуск

### 1. Встановити Node.js 20+

```bash
node -v   # має бути v20+
```

### 2. Встановити залежності

```bash
npm install
```

### 3. Налаштувати БД

Створити PostgreSQL базу даних локально (або використати Docker):

```bash
docker run --name sewing-db -e POSTGRES_PASSWORD=password -p 5432:5432 -d postgres:16
```

Скопіювати `.env.example` в `.env` і вказати реальні значення:

```bash
cp .env.example .env
```

Згенерувати випадковий AUTH_SECRET:

```bash
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
```

### 4. Створити схему БД та адміна

```bash
npx prisma db push
npm run db:seed
```

`db push` створить усі таблиці прямо зі схеми (без файлів міграцій — для MVP цього достатньо).
Адмін створиться з телефоном і PIN з `.env` (за замовчуванням `+380000000000` / `1234`).

### 5. Запустити dev-сервер

```bash
npm run dev
```

Відкрити http://localhost:3000 → ввести телефон і PIN адміна.

## Деплой на Railway

### 1. Залити код в GitHub

```bash
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/USERNAME/sewing-workshop.git
git push -u origin main
```

### 2. Створити проєкт на Railway

1. Зайти на https://railway.app, увійти через GitHub
2. **New Project → Deploy from GitHub repo** → вибрати репозиторій
3. **+ New → Database → Add PostgreSQL** в тому ж проєкті
4. У сервісі застосунку (не БД) → **Variables** додати:
   - `DATABASE_URL` = `${{Postgres.DATABASE_URL}}` (Railway підставить автоматично через reference)
   - `AUTH_SECRET` = ваш випадковий рядок 32+ символи
   - `ADMIN_PHONE` = ваш телефон у форматі `+380...`
   - `ADMIN_PIN` = тимчасовий PIN
   - `ADMIN_NAME` = ваше ім'я
5. **Settings → Networking → Generate Domain**
6. При першому запуску `npm run start` автоматично створить схему БД через `prisma db push`

### 3. Створити першого адміна

Після успішного білду — Railway → ваш сервіс → **... → Run a one-off command**:

```bash
npm run db:seed
```

Готово. Заходимо на згенерований домен, входимо телефоном і PIN-ом з env-змінних.

### 4. Створення робочих швей

Після першого входу адміна:
- **Швеї → + Додати швею** — вводиш ім'я, телефон, PIN (можна згенерувати)
- Передаєш швеї телефон і PIN — вона заходить зі свого мобільного

## Переїзд на власний хостинг (Hetzner / DigitalOcean / тощо)

Для цього є готовий `Dockerfile`. Алгоритм:

1. Орендувати VPS (Hetzner CX22 — €4/міс достатньо для цього проєкту)
2. Поставити Docker і docker-compose
3. Створити `docker-compose.yml` з сервісами `app` (з нашого Dockerfile) і `postgres`
4. Налаштувати **Caddy** або **nginx** як reverse proxy з безкоштовним SSL від Let's Encrypt
5. Експортувати дані з Railway (`pg_dump`) і імпортувати в новий Postgres

Можу скласти детальний гайд по переїзду коли дійдеш до цього кроку.

```
      models/           # CRUD моделей

      payments/         # Прийом робіт + виплати
      fabrics/          # Облік тканин

      analytics/        # Аналітика собівартості
    seamstress/         # Кабінет швеї (мобайл)
      page.tsx          # Головна — 3 великі кнопки
      take/             # Взяти крій (модель → розмір → к-ть)
      works/            # Мої роботи (здати)
      earnings/         # Мій заробіток
    api/
      auth/             # login / logout
      admin/            # CRUD ендпоінти
      seamstress/       # available / take / complete

  lib/
    prisma.ts           # Prisma singleton
    auth.ts             # JWT сесії
    utils.ts            # Format helpers (₴, телефон, тиждень)

  components/           # UI-компоненти
  middleware.ts         # Захист маршрутів
```

## Робочий процес

### Створення моделі
**Адмін → Моделі → + Нова модель** — артикул, назва, фото, розміри, ціни (відшив, крій, тканини м/од.), дод. послуги.

### Додавання крою
**Адмін → Додати крій** — вибрати модель з історії або створити нову → ввести кількості по розмірах → зберегти. Партія записується з повним залишком.

### Взяття крою швеєю
Швея заходить → **Взяти крій** → плитка моделей з фото → тапає модель → тапає розмір → +/− кнопками вказує кількість → **Взяти**. Залишок зменшується атомарно.

### Здача роботи
Швея → **Мої роботи** → кнопка **Здати** → робота переходить в статус «очікує прийому».

### Прийом і виплата
Адмін → **Виплати** → бачить список зданих робіт → **Прийняти** (нараховує суму = ціна відшиву × кількість) або **Брак** (повертає швеї з причиною).

В п'ятницю → секція «Тиждень X-Y» → по кожній швеї **Виплатити** → робиться запис у `Payment`, всі прийняті роботи цього тижня переходять у `PAID`, лічильник «до виплати» обнуляється.

### Аналітика
**Адмін → Аналітика** — за вибраний період бачиш скільки відшито по кожній моделі, повну розбивку собівартості (відшив + крій + тканина + дод. послуги + накладні витрати) і собівартість одиниці.
